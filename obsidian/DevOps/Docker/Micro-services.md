Microservices is a way of building, deploying and managing each application feature as it own small app. Each microservices runs on its own containers
- Db
- logging
- ui

## How to containerize microservices

If we have that many containers, it becomes very clunky very fast to build, containerize and register each container. So we use a declarative approach.

> [!INFO] Declarative means describe an application in a configuration file, and use that file to deploy the application
